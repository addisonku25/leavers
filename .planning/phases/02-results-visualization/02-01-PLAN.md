  <files>src/lib/sankey-data.ts, src/lib/__tests__/sankey-data.test.ts</files>

    - groupMigrationsForCards groups migrations by destinationCompany with totalCount
    - groupMigrationsForCards sorts companies by totalCount descending
    - groupMigrationsForCards sorts roles within each company by count descending
    - groupMigrationsForCards includes seniority comparison for each role
    - buildSankeyData produces nodes with category: "source" | "company" | "destination"
    - buildSankeyData limits to top 8 companies, groups remainder into "Other (X companies)"
    - buildSankeyData limits to top 5 destination roles per company in Sankey, groups remainder into "Other roles"
    - buildSankeyData returns empty nodes/links for empty input (no crash)
    - No individual names appear in any output structure (PRIV-01)
  </behavior>
  <action>
    1. Create src/lib/sankey-data.ts with two main exports:

    **groupMigrationsForCards(migrations, searchedRole):**
    - Input: array of migration records (with optional sourceRole) + the user's searched role string
    - Group by destinationCompany using a Map
    - For each company: sum counts for totalCount, map roles with seniority via compareSeniority(searchedRole, sourceRole)
    - Sort companies by totalCount descending, roles within each company by count descending
    - Return `CompanyCardData[]` (define and export this type)
    - Type: `{ company: string; totalCount: number; roles: { role: string; count: number; seniority: SeniorityComparison }[] }`

    **buildSankeyData(migrations, searchRole):**
    - Input: same migration records + search role
    - Build 3-column Sankey: source role (left) -> destination companies (middle) -> destination roles (right)
    - Source node: the searched role (single node on left)
    - Company nodes: top 8 by total count as individual nodes, remainder grouped into "Other (X companies)" node
    - Destination role nodes: top 5 per company, remainder grouped into "Other roles" per company
    - Links: source->company (value = total migration count to that company), company->destRole (value = migration count for that role)
    - Return `SankeyData` type: `{ nodes: SankeyNode[]; links: SankeyLink[] }` where `SankeyNode = { name: string; category: "source" | "company" | "destination" }` and `SankeyLink = { source: number; target: number; value: number }`
    - Guard: if input is empty array, return `{ nodes: [], links: [] }`

    2. Create src/lib/__tests__/sankey-data.test.ts testing:
    - Grouping: 5 migrations across 3 companies -> 3 cards sorted by count
    - Role sorting: roles within a card are sorted by count descending
    - Seniority: roles include correct seniority comparison
    - Sankey top-8: 10 companies -> 8 individual + "Other (2 companies)" node
    - Sankey top-5 roles: company with 7 roles -> 5 individual + "Other roles" node
    - Empty input: returns empty arrays, no crash
    - Privacy (PRIV-01): output contains no fields named "name", "email", "linkedin", or "profile" -- only company names, role titles, and counts
  </action>
