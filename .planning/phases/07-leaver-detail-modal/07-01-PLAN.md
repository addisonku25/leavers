```

```

Define response types:
- `LeaverPositionData`: { company, title, startDate, endDate }
- `PublicLeaver`: { id, currentTitle, currentCompany, transitionDate, positions: LeaverPositionData[] }
- `AuthenticatedLeaver extends PublicLeaver`: adds { name, linkedinUrl }
- `LeaverModalData`: { isAuthenticated, leavers, totalCount }

Export the response types so Plan 02 can import them.

Query flow:
1. Check session via `auth.api.getSession`
2. Query `leavers` table WHERE migrationId matches
3. For each leaver, query `leaverPositions` WHERE leaverId matches, ORDER BY sortOrder ASC
4. If authenticated: return full leaver objects including name + linkedinUrl
5. If unauthenticated: return PublicLeaver objects WITHOUT name/linkedinUrl fields (do NOT include them as null -- omit entirely per PRIV-05)

For tests: mock `auth.api.getSession` (return session object or null), mock `db` queries. Use vitest `vi.mock` for `@/lib/auth` and `@/lib/db`.
