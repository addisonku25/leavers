<feature>
  <name>Insights computation engine (TDD)</name>
  <files>src/lib/insights.ts, src/lib/__tests__/insights.test.ts</files>

  <behavior>
    INSI-01 - computeTopDestinations:
    - Given migrations with 3 companies (counts: 10, 5, 3), returns them ranked with percentages [56%, 28%, 17%]
    - Given migrations with 8+ companies, returns only top 5
    - Given empty migrations array, returns empty array
    - Given single company, returns [100%]
    - Percentages are rounded integers (Math.round)

    INSI-02 - classifyRole + computeRoleBuckets:
    - "VP of Sales" classifies as Leadership (not Business, even though "Sales" matches Business)
    - "Software Engineer" classifies as Technical
    - "Account Executive" classifies as Business
    - Given searchedRole "Solutions Engineer", destination "Solutions Engineer" classifies as "Same role"
    - Given searchedRole "Solutions Engineer", destination "Sr. Solutions Engineer" classifies as "Same role" (normalized match)
    - "Photographer" (no keyword match) defaults to Business
    - computeRoleBuckets returns only non-empty buckets, sorted by percentage descending
    - Each bucket includes top 2-3 specific role names

    INSI-03 - generatePatternSummary:
    - When top destination has >= 25% concentration, includes concentration sentence
    - When "Same role" bucket exists, includes role change frequency sentence
    - Always includes top transition highlight sentence
    - Includes seniority trend when seniority data is available (using parseSeniorityLevel on sourceRole vs destinationRole)
    - Returns 2-3 sentences joined as a single string
    - Sentences are ordered by "notability" score (most interesting pattern first)

    computeInsights (orchestrator):
    - Returns InsightsData with all three sections populated
    - Handles empty migrations gracefully (empty topDestinations, empty roleBuckets, empty patternSummary)
    - Handles very small result sets (<3 migrations) with reduced/caveat output

    Edge cases:
    - Division by zero: totalMigrations=0 produces no NaN
    - All migrations to same company: 100% concentration
    - All same role: "Same role" bucket is 100%
  </behavior>

  <implementation>
    Create src/lib/insights.ts with these exports:

    Types:
    - TopDestination: { company: string; percentage: number }
    - RoleCategory: "Leadership" | "Business" | "Technical" | "Same role"
    - RoleBucket: { category: RoleCategory; percentage: number; topRoles: string[] }
    - InsightsData: { topDestinations: TopDestination[]; roleBuckets: RoleBucket[]; patternSummary: string; totalMigrations: number }

    Constants (named, tunable):
    - CONCENTRATION_THRESHOLD = 25 (% for "dominant destination")
    - MIN_MEANINGFUL_MIGRATIONS = 3 (below this, show caveat)
    - MAX_TOP_DESTINATIONS = 5

    Functions:
    1. classifyRole(destinationRole: string, searchedRole: string): RoleCategory
       - Check "Same role" first via normalizeRoleTitle comparison
       - Then Leadership regex (VP, Director, Head of, Chief, etc.)
       - Then Technical regex (Engineer, Developer, Architect, SRE, etc.)
       - Then Business regex (PM, Sales, AE, Success, Marketing, etc.)
       - Default: "Business" as catch-all per research recommendation

    2. computeTopDestinations(migrations: MigrationRecord[], limit?: number): TopDestination[]
       - Aggregate by destinationCompany, compute percentages, sort descending, take top N

    3. computeRoleBuckets(migrations: MigrationRecord[], searchedRole: string): RoleBucket[]
       - Classify each migration's destinationRole, aggregate by bucket, compute percentages
       - Each bucket gets top 2-3 role names (by count within bucket)
       - Filter out 0% buckets, sort by percentage descending

    4. generatePatternSummary(topDestinations: TopDestination[], roleBuckets: RoleBucket[], migrations: MigrationRecord[], searchedRole: string): string
       - Detect 4 pattern types, score each by "notability", sort by score, take top 2-3
       - Concentration: score = top destination percentage (if >= CONCENTRATION_THRESHOLD)
       - Role change: score based on "Same role" bucket size (interesting when very high or very low)
       - Top transition: always included, score = count of most common source->dest pair
       - Seniority trend: use parseSeniorityLevel on sourceRole vs destinationRole across migrations
       - Join selected pattern sentences into natural paragraph

    5. computeInsights(migrations: MigrationRecord[], searchedRole: string): InsightsData
       - Orchestrates the above, handles empty/small arrays with early return
  </implementation>
</feature>
