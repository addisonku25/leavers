**leaver-timeline.tsx** -- Pure presentational component:
- Props: `{ positions: LeaverPositionData[] }` (imported from `src/actions/leavers.ts`)
- Renders a vertical timeline with a `div` for the connecting line (absolute positioned, left side)
- Each position is a node with:
  - Filled circle (`bg-primary border-primary`) for index 0 (most recent/current)
  - Open circle (`bg-background border-muted-foreground`) for past positions (index > 0)
  - Circle size: `size-3 rounded-full border-2`
  - Content: `{title} @ {company}` in `text-sm font-medium`, date range in `text-xs text-muted-foreground`
- Date range formatting: `{startDate} - {endDate ?? "Present"}`. If both null, omit date line. Format dates as-is (they're already strings like "2024-03" or "Jan 2024")
- Use `cn()` for conditional classes
- Spacing: `pl-6` for content area, nodes have `pb-4 last:pb-0`

**auth-gate-overlay.tsx** -- Frosted glass CTA overlay:
- Props: `{ returnTo: string }` (the current page path for redirect after signup)
- Renders a `div` with `backdrop-blur-md bg-background/60` overlay
- Contains centered content: Lock icon (lucide-react), heading "Sign up to see all profiles", description text explaining value, and a Button linking to `/signup?returnTo={returnTo}`
- Use `Link` from next/link for the CTA button (styled as primary Button with `asChild`)

**leaver-modal.tsx** -- Dialog wrapper and content orchestrator:
- Props: `{ open: boolean; onOpenChange: (open: boolean) => void; migrationId: string | null; role: string; company: string; count: number; returnTo: string }`
- Uses `Dialog` from `@/components/ui/dialog` with `open` and `onOpenChange` props (controlled mode)
- On open (when `migrationId` changes and is non-null), call `getLeaversForMigration(migrationId)` via `useEffect` + `useState`
- Include race condition protection: compare migrationId at time of fetch with current migrationId before setting state (per Research Pitfall 2)
- Loading state: Show a spinner or skeleton while fetching
- Empty state: "No individual data available for this transition" message when 0 leavers returned

**Modal content layout (per CONTEXT.md locked decisions):**
- `DialogContent` with `className="max-h-[85vh] overflow-y-auto sm:max-w-lg"` plus mobile sheet overrides: `max-sm:fixed max-sm:bottom-0 max-sm:left-0 max-sm:right-0 max-sm:top-auto max-sm:translate-x-0 max-sm:translate-y-0 max-sm:w-full max-sm:max-w-full max-sm:rounded-b-none max-sm:rounded-t-xl`
- `DialogHeader` with `DialogTitle`: "{role} @ {company}" and `DialogDescription`: "{count} people made this transition"
- Leaver list section:
  - Use `authClient.useSession()` for client-side rendering decisions (show name vs blur)
  - Show first 3 leavers by default. If more exist, show "Show N more" button that reveals the rest (useState toggle)
  - For each visible leaver:
    - If authenticated: Show leaver name (text-base font-semibold), LinkedIn link (ExternalLink icon, `target="_blank" rel="noopener noreferrer"`), and `LeaverTimeline` with positions
    - If unauthenticated AND index === 0: Show blurred placeholder for name (`blur-sm select-none` on a span containing "Full Name Available"), hide LinkedIn link, show `LeaverTimeline` with full career data (career data is NOT PII)
    - If unauthenticated AND index > 0: These leavers are behind the `AuthGateOverlay` -- render the overlay component positioned over the remaining leavers area
  - The auth gate overlay should wrap/cover the leavers at index 1+ using relative positioning on the container and absolute positioning on the overlay

**Important per CONTEXT.md:** Opening modal does NOT change DrillDownProvider state. Modal state is local -- do not import or dispatch to useDrillDown.
