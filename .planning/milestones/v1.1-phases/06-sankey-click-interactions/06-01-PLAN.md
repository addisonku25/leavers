       - If `node.category === "source"`: dispatch `{ type: "CLEAR" }`
       - If `node.category === "company"`: check if already selected (same value) -- if so dispatch CLEAR, else dispatch `{ type: "SELECT_COMPANY", company: node.name, nodeIndex }`
       - If `node.category === "destination"`: check if already selected (same value) -- if so dispatch CLEAR, else dispatch `{ type: "SELECT_ROLE", role: node.name, nodeIndex }`

    5. Update opacity helpers to layer hover + selection:
       - Create `getActiveHighlightNode(): number | null` that returns `hoveredNode` if non-null, else `selectedNode` if non-null, else null
       - Create `activeConnectedNodes` useMemo that computes connected nodes from `getActiveHighlightNode()` result (reuse existing connectedNodes logic but make it work for both hover and selection)
       - IMPORTANT: Do NOT add `selectedNode`/`drillDown` to the Sankey layout `useMemo` deps -- selection only affects rendering, not d3 layout computation (Pitfall 1 from research)

    6. Add stroke to selected node's `<rect>`:
       - `stroke={selectedNode === i ? "hsl(221, 83%, 40%)" : "none"}`
       - `strokeWidth={selectedNode === i ? 2 : 0}`
       - Add `className="transition-[stroke] duration-200"` for smooth stroke appearance

    7. Add `onClick={() => handleNodeClick(i, node as unknown as SankeyNode)}` to each node's `<g>` element (alongside existing onMouseEnter)

    8. Keep existing `onMouseLeave={() => setHoveredNode(null)}` unchanged -- the layered approach means when hover clears, selection visual automatically restores through the `getActiveHighlightNode` logic
